1. Software Requirements Specification (SRS)

1.1 Pendahuluan

Sistem KDKMP bertujuan untuk menyatukan pencatatan keuangan desa dan aktivitas barter komoditas desa secara digital. Melalui sistem ini, desa memiliki database terpusat yang aman dengan hak akses (RBAC) yang terbagi jelas.

1.2 Aktor & Hak Akses

Aktor/Role Hak Akses di Sistem Tanggung Jawab
Keuangan CRUD Transaksi Keuangan, CRUD Termin, Generate Reports Mencatat aliran kas desa, mengatur termin cicilan proyek desa, memonitor laporan keuangan.
Barter CRUD Komoditas Sendiri, Pencatatan Barter Internal, Riwayat Barter Memperbarui jumlah stok hasil bumi desa (komoditas), melakukan transaksi pencatatan barter internal.

2.2 Laporan Progres Proyek (Awal s.d. Akhir)

Tahap Awal (Inisiasi & Perancangan):

Proyek dimulai dengan memetakan kebutuhan sistem komoditas desa. Kami merancang skema relasi antarentitas. Julia memimpin analisis SRS sementara Aldi menyiapkan repositori git awal dan mengonfigurasikan lingkungan database local MySQL.

Tahap Tengah (Implementasi Kode & Database):

Kami beralih menggunakan database MySQL. Database seeder dibuat untuk memudahkan pengisian data dummy. Aldi merancang struktur visual template Blade. Julia mengimplementasikan controller backend dengan validasi kepemilikan (ownership) dan kebijakan keamanan (policy).

Tahap Akhir (Polishing, Testing, & Launch):

Pada tahap akhir, kami mengintegrasikan style UI agar seragam dan responsif. Pengujian dijalankan dengan menulis Automated Feature Tests pada sistem autentikasi dan otorisasi role untuk memastikan sistem bebas dari bug akses ilegal.
